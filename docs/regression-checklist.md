# GreenSerial 回归清单

更新时间：2026-03-26

## 1. 硬件联机回归（真实串口）

### 1.1 环境探测记录
- 命令：`Get-CimInstance Win32_SerialPort | Select-Object DeviceID,Name,PNPDeviceID`
- 命令：`[System.IO.Ports.SerialPort]::GetPortNames()`
- 历史记录（第一次）：未检测到可用物理串口（空结果）
- 最新记录（硬件接入后）：检测到 `COM5`

### 1.1.1 本次实测结果（COM5）
- 只读冒烟：打开成功，读取到 `00`（1 字节），关闭成功。
- 写入冒烟：发送 `55 AA 01 02` 成功，2 秒窗口内未收到回包，关闭成功。
- 说明：当前设备可能不是回显设备，或需特定协议命令才返回数据。

### 1.1.2 波特率快速探测（COM5）
- 只读探测（`9600/19200/38400/57600/115200/230400/460800`）：均可打开端口，窗口内无自动上报码。
- 写入探测（`9600` 与 `115200`，发送 `55 AA 01 02`）：写入成功，窗口内无回包。
- 建议：使用设备协议中的“查询状态/读寄存器”有效命令再做回归，以确认 RX 链路。

### 1.1.3 Zynq 登录探测（root/root）
- 脚本：`scripts/zynq_login_smoke.ps1`
- 测试参数：`COM5 @ 115200`，用户名/密码 `root/root`，普通模式 + 强制发送凭据模式。
- 测试结果：串口始终无任何文本输出（未检测到 `login:`、`Password:` 或 shell 提示符）。
- 额外抓包：`scripts/zynq_console_capture.ps1`（普通模式 + `DTR/RTS` 开启）均无输出。
- 结论：当前主机侧串口可打开可写，但未接收到 Linux 控制台文本；建议确认板卡 UART console 路由与 `bootargs` 的 `console=ttyPSx,115200` 配置。

### 1.1.4 常见现象说明（U-Boot）
- 日志出现 `*** Warning - bad CRC, using default environment`：
  这通常表示 U-Boot 环境区校验失败，已回退默认环境；不是串口链路故障本身。
- 日志停在 `Zynq>`：
  表示当前处于 U-Boot 命令行，Linux 尚未启动，所以不会出现 `login:`。
- 在 `Zynq>` 下建议先执行：
  1. `printenv bootcmd`
  2. `run bootcmd`
  3. 若仍无法进 Linux，再检查启动介质与内核镜像加载路径（SD/NAND/QSPI）。

### 1.1.5 已进 Ubuntu 但串口无回显时
- 现象：板端已登录 root，但串口工具无法稳定读取命令回显。
- 处理：直接在板端运行本地采集脚本（不依赖串口交互）：
  1. 将 `scripts/zynq_collect_sysinfo.sh` 放到板子（如 `/tmp`）。
  2. 执行：`chmod +x zynq_collect_sysinfo.sh`
  3. 执行：`./zynq_collect_sysinfo.sh /tmp`
  4. 产物：`/tmp/zynq_sysinfo_时间戳` 与同名 `.tar.gz`

### 1.2 回归步骤（有硬件时执行）
预检查脚本：

`powershell -ExecutionPolicy Bypass -File scripts/hardware_serial_smoke.ps1`

1. 启动应用并进入终端页，点击“扫描串口”。
2. 选择真实串口，波特率设为设备默认值，点击“连接设备”。
3. 观察状态标签应变为“已连接”。
4. 在发送面板输入 ASCII 文本并发送，确认设备端收到数据。
5. 切换为 HEX 模式发送（例如 `AA 55 01 02`），确认设备端收到字节流。
6. 设备端回传数据后，确认日志面板出现 `RX` 记录。
7. 进入波形页，点击“开始”，确认画布有实时曲线变化。
8. 切换页面（终端/预设/设置）再返回，确认连接与日志仍保持。
9. 点击“断开连接”，确认状态恢复为“未连接”且不可再发送。

### 1.3 验收标准
- 能稳定连接、发送、接收、断开。
- RX 数据能同步驱动日志与波形。
- 切页不丢连接状态，手动断开后状态一致。

## 2. 无硬件模拟回放回归

### 2.1 功能入口
- 当扫描不到物理串口时，连接列表自动提供 `MOCK://ReplayPort`（模拟回放端口）。
- 连接该端口后会持续生成模拟 RX 数据，驱动日志和波形。

### 2.2 回归步骤
1. 扫描串口，确认可见 `MOCK://ReplayPort`。
2. 连接该端口，状态应为“已连接”。
3. 日志面板应持续出现 `RX` 数据。
4. 波形页点击“开始”，曲线应持续更新；暂停后停止刷新。
5. 在发送面板发送 ASCII 与 HEX 数据，日志应记录 `TX`，并收到模拟响应 `RX`。
6. 在预设页发送预设命令，日志应记录 `TX` + `RX`。
7. 点击“断开连接”，模拟数据应停止增长。

### 2.3 验收标准
- 无硬件情况下可完整覆盖连接/发送/接收/波形链路。
- 模拟模式下发送行为与真实模式交互一致。
- 断开后无残留定时回放任务。
